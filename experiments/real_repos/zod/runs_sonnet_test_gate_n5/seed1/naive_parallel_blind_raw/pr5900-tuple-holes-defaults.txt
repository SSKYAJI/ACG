Now let me write the actual fix. Based on the Zod v4 source code structure, the `parseTuple` function in `schemas.ts` has a loop that skips optional items when `i >= input.length`. The bug is likely that when we have `[optional, required_with_default]` and input is `[undefined]`, the `lastRequired` is computed as -1 (because the required_with_default item is marked as `optional: true` in the tuple def due to having a default), so the length check passes, and then the loop processes position 0 (undefined, optional - ok) and position 1 (i=1 >= input.length=1, optional=true → skip). This means the required default field is silently skipped.

The fix: when skipping an item because `i >= input.length`, we should still apply the schema (which will apply the default). OR: we should not mark items with defaults as `optional` in the tuple def, but instead handle them differently.

Actually, looking at this more carefully: if `lastRequired = -1` because all items are marked optional (including those with defaults), then `input.length < -1 + 1 = 0` is never true, so any length passes. Then in the loop, items beyond `input.length` are skipped entirely - including those with defaults. This means the default is never applied!

The fix should be: when `i >= input.length` and the item is optional (has a default), we should still run the schema with `undefined` as input (which will trigger the default), rather than skipping it entirely.
