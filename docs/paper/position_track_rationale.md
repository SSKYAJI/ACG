# Position Track Rationale

This submission is suited to the Position Paper Track because its primary contribution is a normative claim about how parallel coding agents should be designed, not a large-scale empirical benchmark. The paper argues that multi-agent coding systems should use pre-flight write-contract lockfiles to decide safe parallelism before execution, and it uses ACG only as an existence proof that the design is feasible. The empirical material is intentionally limited and is framed as directional support rather than definitive evaluation. That makes the paper a better fit for a venue that values clear positions, architectural arguments, and engagement with competing views. The paper also directly addresses viable counterpositions, including post-hoc merge resolution, runtime CRDT-style coordination, and the view that stronger agents with larger context windows make explicit contracts unnecessary. In short, the paper is advancing a concrete design stance for agentic software engineering, defending that stance against plausible alternatives, and using a lightweight prototype plus small-scale evidence to show that the position is actionable now. That is exactly the kind of contribution the Position Paper Track is designed to surface.
