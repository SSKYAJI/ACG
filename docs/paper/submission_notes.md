# Submission Notes

## Main Paper

- Keep the main submission within the conference's 9-page limit.
- Preserve double-blind review by omitting author names, affiliations, and direct self-identifying repository links from the main paper.
- Keep the title, abstract, and introduction thesis sentence explicit about the position.
- Keep the `Alternative Views` section in the main body, not in an appendix.
- Frame all quantitative evidence as small-scale directional support, not as benchmark-quality evaluation.

## Recommended Methodology / Discipline Tags

- Software engineering
- Static analysis
- Multi-agent systems
- Programming tools / developer infrastructure
- Human-AI programming systems

## Pre-Submission Cleanup

- Move the draft into the venue template once the target conference style file is chosen.
- Convert inline URLs in the references into the template's bibliography format.
- Verify every non-CodeCRDT citation before submission if any exact wording from those sources is used.
- Trim or expand sections to match the final page count after the template is applied.
